# -----------------------------------------------------------------------------
# §1 — Test framework
# -----------------------------------------------------------------------------
# v2.1 ships ~200+ unit tests across 7 test files. pytest is the
# canonical runner.

pytest>=7.4.0,<9.0.0
# Justification: every `tests/test_*.py` file uses pytest fixtures,
# parametrize markers, and pytest.raises. We require >=7.4 for the
# `tmp_path` fixture's typed return value and improved -m marker
# parsing (used by `pytest -m "not gpu"`).

pytest-cov>=4.1.0
# Justification: produces coverage reports for CI. Used by
# .github/workflows/ci.yml's `pytest --cov=uid_theory` step.

pytest-xdist>=3.5.0
# Justification: parallel test execution via `pytest -n auto`.
# CPU-only test runs benefit ~3-5x speedup; the GPU test suite
# (TestEndToEndGPU in test_energy_meter.py) opts out via @pytest.mark.gpu.

pytest-timeout>=2.2.0
# Justification: prevents stuck tests from blocking CI. The
# `_xi_for_channel` test in test_correlation_length.py and the
# pynvml-backed energy tests can occasionally hang if external
# hardware is unresponsive.

# -----------------------------------------------------------------------------
# §5 — Power measurement (for tests/test_energy_meter.py)
# -----------------------------------------------------------------------------
# OPTIONAL: only needed on machines with NVIDIA GPUs for the
# end-to-end energy benchmark tests (TestEndToEndGPU and the
# real-hardware runs of experiments/run_energy_benchmark.py).

nvidia-ml-py>=12.535.108
# Justification: this is the official Python binding for NVML.
# `uid_theory/verification/energy_meter.py` (v2.1 batch 4) uses
# pynvml.nvmlDeviceGetPowerUsage to achieve 25 Hz power sampling.
# On CPU-only machines, energy_meter.py falls back to the
# nvidia-smi subprocess sampler at 10 Hz max; pynvml is strongly
# recommended for any quantitative energy measurement.


# -----------------------------------------------------------------------------
# §6 — Numerical / scientific (for verification suite tests)
# -----------------------------------------------------------------------------
# These are runtime deps that the verification suite needs at
# elevated versions for reproducibility.

scipy>=1.11.0
# Justification: required by
#   - `uid_theory/verification/critical_exponents.py` (uses
#     scipy.stats.linregress, scipy.fft.rfft, scipy.special.digamma)
#   - `uid_theory/verification/correlation_length.py` (uses
#     scipy.spatial.cKDTree for the KSG MI estimator,
#     scipy.special.digamma for the KSG correction term)
#   - `uid_theory/verification/powerlaw_estimator.py` (uses
#     scipy.stats.norm for the LR test p-value).
# v1.11 is the first version with the stable scipy.fft module that
# v2.1 relies on; earlier versions used scipy.fftpack (deprecated).

numpy>=1.26.0
# Justification: the numerical foundation. v1.26 is the first version
# with the stable np.random.default_rng() API used throughout
# the v2.1 test suite. Avoid v2.0 for now — it dropped
# `np.float128` which some downstream tools still reference.

# -----------------------------------------------------------------------------
# §12 — End notes
# -----------------------------------------------------------------------------
#
# Pinning strategy:
#   - Direct deps pinned with `>=` minimum versions reflecting v2.1
#     API requirements.
#   - No upper bounds, except where a known breaking release exists
#     (e.g. numpy 2.0 may break np.float128 consumers downstream).
#
# Reproducibility:
#   - For exact reproduction of Phase 1 results, see
#     `results/phase1/MANIFEST.json` "environment" block, which
#     pins exact versions used to produce each published result.
#   - This `requirements-dev.txt` is for the DEVELOPMENT environment
#     only, not for reproducing specific published numbers.
#
# CI compatibility:
#   - Tested against Python 3.10, 3.11, 3.12 on Ubuntu 22.04.
#   - GPU tests additionally require CUDA 12.1+ and a compatible
#     PyTorch wheel (the `torch` line in requirements.txt is
#     intentionally unpinned for CUDA version, so users must install
#     the appropriate wheel via the PyTorch install matrix:
#     https://pytorch.org/get-started/locally/).
#
# Reporting an environment problem:
#   - File an issue at https://github.com/gwailee/uid/issues with the
#     `environment` label, including the output of `pip freeze` and
#     the failing test name.
#
